Send a message on WhatsApp: "Summarize my unread emails and schedule a meeting with Sarah for Thursday afternoon."

  1. Understands your intent via natural language processing

Architecture & High-Level Design

üèõÔ∏è Component 1: The Gateway

The Gateway is the heart of OpenClaw ‚Äî think of it as an air traffic control tower. It's a WebSocket control plane running on port 18789 by default.

üîó

Connection Hub

Maintains WebSocket connections to all channels and routes messages

üìã

Session Management

Creates isolated sessions per conversation with full history

‚öôÔ∏è

Configuration

Reads ~/.openclaw/openclaw.json, hot-reloads changes

üåê

Web Interface

Serves Control UI at /__openclaw__/

bash ‚Äî key gateway commands
openclaw gateway --port 18789   # Start the gateway
openclaw gateway status          # Check status
openclaw logs --follow           # View logs in real-time
openclaw config apply            # Hot-reload config

üß† Component 2: The Pi Agent Runtime

Pi is OpenClaw's embedded agent runtime (RPC mode). This is where the "intelligence" lives. It handles prompt building, LLM invocation, response parsing, and tool orchestration in a streaming loop.

  • Streaming responses (real-time output)
  • Block streaming (structured content like code, tables)
  • Tool orchestration (chain multiple tools together)
  • Multi-turn reasoning (complex problem solving)

Step 1: Install Node.js

bash ‚Äî macOS/Linux
brew install node        # macOS via Homebrew

Step 2: Install OpenClaw

3.2 Configuration

json ‚Äî ~/.openclaw/openclaw.json
{
  "gateway": {
    "port": 18789,
    "bind": "loopback",
    "auth": { "token": "your-secure-token-here" }
  },
  "models": {
    "default": "anthropic/claude-4-opus",
    "thinking": "medium"
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "${TELEGRAM_BOT_TOKEN}",
      "dmPolicy": "pairing"
    }
  }
}

Setting Up Telegram

  1. Message @BotFather on Telegram, send /newbot, copy the token
  2. Run openclaw config set channels.telegram.botToken "YOUR_TOKEN"
  3. Start the gateway: openclaw gateway --port 18789 --verbose
  4. Approve the first contact: openclaw pairing approve telegram <code>

Security: OpenClaw uses pairing by default. Unknown users must be explicitly approved before receiving responses.
